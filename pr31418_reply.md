The warning is being silently swallowed. `pytest.ini` sets `filterwarnings = error`, which turns the `DeprecationWarning` into an exception. But `recarray.__setattr__` has a broad `except Exception:` (the field-name fallback path that routes `ra.shape = ...` to `setfield('shape', ...)` when the direct attribute set raises), and it catches the converted warning right along with the legitimate `TypeError` we expect for an invalid shape. So the test passes under pytest CI. The warning only leaks in wheel-build runs where `filterwarnings = error` isn't applied.
